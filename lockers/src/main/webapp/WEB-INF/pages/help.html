Lockers overview

This section explains the usage of the lockers overview page.

The locker overview is the main page. You can quickly search for lockers and assign or clear users from these lockers.

Search

The search fields will search all the columns in the table. It does so using regular expressions, so it's possible to search with more complex terms. An example would be [a,c][1-3]0[1-3]. This gets all lockers with towers 'A' and 'C', on floors '1, 2, 3' with numbers '01, 02, 03'. 'A101' would fit these requirements.

Seperated by a space, you could also search for lockers in any order of value. Example: 01 1 A would return the A101 locker.

Remember: the regular expression is quite often case sensitive! For example: [a|b]\d0[1-3] will not get any locker, because all the towers are upper case. The correct query here would be [A|B]\d0[1-3]

Locker detail

The locker detail page is split up in three parts:

  • Locker information

    The basic information (Tower, Floor and Number) show the location of the locker. 'Acquired' indicates when the current user was assigned the locker. If there is no user assigned to the locker, this field indicates when the user was removed. 'Date expired' is the date on which the locker should no longer belong to the user.

  • User information

    All the information about the user which is assigned to the locker can be found here. A user can also be assigned or cleared here. These two actions work the same as on the locker overview page

  • History

    Up to the most recent 100 actions of this specific locker will be displayed here.
